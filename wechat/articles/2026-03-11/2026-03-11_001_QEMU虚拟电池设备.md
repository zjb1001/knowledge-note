## 📌 文章概述

本文介绍如何通过修改 QEMU 的 **SMBIOS** 和 **ACPI Table** 让虚拟机中的操作系统（Windows/Linux）识别出虚拟电池设备，使虚拟机表现得像真实的笔记本电脑一样具有电池功能。

---

## 一、技术原理

### 1.1 双管齐下：SMBIOS + ACPI

| 组件 | 作用 | 提供的信息类型 |
|------|------|----------------|
| **SMBIOS** | 提供静态硬件信息 | 电池厂商、设计容量、序列号、设备名称等 |
| **ACPI Table (DSDT)** | 提供动态状态与控制 | 实时电量、充放电状态、电压等 |

## 二、SMBIOS 配置（静态信息）

### 2.1 Type 22 Structure

SMBIOS 中的 **Portable Battery (Type 22)** 结构用于描述电池的物理属性：

```c
// SMBIOS Type 22 Structure 定义
struct smbios_type_22 {
    uint8_t type;           // 22 = Portable Battery
    uint8_t length;         // 结构长度
    uint16_t handle;        // 唯一标识
    uint8_t location;       // 电池位置字符串
    uint8_t manufacturer;   // 制造商字符串
    uint8_t manufacture_date;
    uint8_t serial_number;  // 序列号字符串
    uint8_t device_name;    // 设备名称字符串
    uint8_t device_chemistry;
    uint16_t design_capacity;
    uint16_t design_voltage;
    uint8_t sbds_version;
    uint8_t max_error;
    uint16_t sbds_serial_number;
    uint16_t sbds_manufacture_date;
    uint8_t sbds_device_chemistry;
    uint8_t design_capacity_mult;
    uint32_t oem_specific;
};
```

### 2.2 实际配置示例

```c
// QEMU 代码中设置的电池信息
Manufacturer: "MyStackTrace"
Serial Number: "123456789"
Design Capacity: 0xFFFF (100% 设计容量)
```

## 三、ACPI 配置（动态状态）

### 3.1 电池设备节点

ACPI 命名空间中电池设备的 **Hardware ID** 为 `PNP0C0A`：

```asl
// DSDT 中的电池设备定义 (ASL 语言)
Device (BAT0) {
    Name (_HID, EisaId ("PNP0C0A"))  // 电池设备标识
    
    // _STA: 设备状态方法
    Method (_STA, 0, NotSerialized) {
        Return (0x1F)  // 设备存在且可用
    }
    
    // _BIF: 电池信息方法
    Method (_BIF, 0, NotSerialized) {
        Return (Package (0x0D) { ... })  // 13个字段
    }
    
    // _BST: 电池状态方法
    Method (_BST, 0, NotSerialized) {
        Return (Package (0x04) { ... })  // 4个字段
    }
}
```

### 3.2 _BIF 方法（Battery Information）

返回 **Package** 包含 13 个字段，提供电池静态信息：

| 索引 | 字段 | 说明 |
|------|------|------|
| 0 | Power Unit | 电量单位 (mWh 或 mAh) |
| 1 | Design Capacity | **设计容量** - 电池出厂标称的理想最大容量 |
| 2 | Last Full Charge | **最后一次满充容量** - 实际可容纳的最大电量 |
| 3 | Battery Technology | 技术类型 (可充电/不可充电) |
| 4 | Design Voltage | 设计电压 |
| 5 | Design Capacity of Warning | 低电量警告阈值 |
| 6 | Design Capacity of Low | 低电量临界阈值 |
| 7-12 | Cycle Count, Measurement, etc | 其他信息 |

**关键概念**:
- **Design Capacity**: 电池新买时应该能存多少电
- **Last Full Charge**: 电池现在最多能存多少电（体现老化程度）

### 3.3 _BST 方法（Battery Status）

返回 **Package** 包含 4 个字段，提供电池动态状态：

| 索引 | 字段 | 说明 |
|------|------|------|
| 0 | **Battery State** | 状态标志位（充放电状态） |
| 1 | Battery Present Rate | 当前充放电速率 |
| 2 | **Battery Remaining** | **当前剩余容量** |
| 3 | Battery Present Voltage | 当前电压 |

**Battery State 标志位**:
- `0x00`: 正常放电
- `0x01`: 放电中
- `0x02`: 充电中
- `0x04`: 严重警告
- `0x05`: 充电完成

### 3.4 电量计算公式

```
电量百分比 = (Battery Remaining / Last Full Charge) × 100%
```

**示例**:
- `Battery Remaining = 0x1000`
- `Last Full Charge = 0xFFFF`
- `电量 = (0x1000 / 0xFFFF) × 100% ≈ 6%`

## 八、扩展思考

### 8.1 当前局限

- 电池状态是 **hard code**（硬编码）的，不会动态变化
- 无法实现真实的充放电模拟

### 8.2 可能的改进方向

| 改进方向 | 实现思路 |
|----------|----------|
| 动态电量变化 | 在 QEMU 中实现定时器，周期性更新 _BST 返回值 |
| 充电/放电切换 | 监听电源事件，修改 Battery State 标志位 |
| 外部控制接口 | 添加 QMP/QEMU Monitor 命令，允许外部修改电池状态 |
| 多电池支持 | 创建 BAT1, BAT2 等多个设备节点 |
| 电池老化模拟 | 随时间递减 Last Full Charge 值 |
