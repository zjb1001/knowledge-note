# HIL 测试环境搭建规划

**规划目标**: 为 BrakeGuard 制动控制器搭建硬件在环(HIL)测试环境  
**规划日期**: 2026-03-14  
**规划者**: CEO产品规划专家（GStack /plan）

---

## 核心洞察

**一句话概括**: 搭建一套可复用的 HIL 测试平台，支持制动控制器的闭环仿真测试，实现控制器硬件与车辆动力学模型的实时交互。

**关键约束识别**:
- 制动系统安全关键，HIL测试是量产前的必要环节
- 需要实时性（通常 < 1ms 步长）
- 需要模拟传感器信号（压力、速度、加速度）
- 需要采集控制器输出（PWM、开关信号）
- 成本约束：从 DIY 方案到商用 HIL 设备跨度大

---

## 方案对比

| 方案 | 描述 | 成本 | 工时 | 实时性 | 建议 |
|------|------|------|------|--------|------|
| **MVP** | PC + I/O板卡（如NI USB-6001）+ Python/MATLAB实时脚本 | ¥5k | 2天 | 软实时(~10ms) | **快速验证** - 适合算法验证阶段 |
| **标准版** | 工控机 + dSPACE/NI PXI + Simulink Real-Time | ¥50k-200k | 1周 | 硬实时(<1ms) | **推荐** - 平衡成本与性能 |
| **10星级** | 商用HIL台架（如dSPACE SCALEXIO + 真实执行器） | ¥500k+ | 1月+ | 硬实时(<100μs) | **量产级** - 本次PI不追求 |

## 推荐方案：MVP + 标准版演进

### 阶段一：MVP HIL（第 1-2 天）
**目标**: 快速搭建可验证制动控制逻辑的HIL环境

**硬件清单**:
| 设备 | 型号/规格 | 成本 | 用途 |
|------|----------|------|------|
| 工控PC | i7 + 16GB + SSD | ¥8k | 主机 |
| USB DAQ | NI USB-6001 或国产替代 | ¥2k | 模拟量采集/输出 |
| 信号调理板 | 自制/开源 | ¥1k | 电压匹配、隔离 |
| 制动控制器 | TC3x7 开发板 | 已有 | 被测设备(DUT) |
| 模拟执行器 | 电阻负载 + LED指示 | ¥500 | PWM输出指示 |

**软件栈**:
- **实时引擎**: Python + SimPy 或 MATLAB/Simulink + 快速原型
- **车辆模型**: 简化单轮制动模型（1-DOF）
- **I/O驱动**: NI-DAQmx 或开源库
- **上位机**: Python PyQt 或 MATLAB App Designer

**系统架构**:
```
┌─────────────────┐      ┌──────────────┐      ┌─────────────┐
│   工控PC (主机)  │      │   信号调理    │      │   TC3x7     │
│                 │      │              │      │  (DUT)      │
│ ┌─────────────┐ │      │              │      │             │
│ │ Simulink模型 │ │      │ AI: 压力信号 │      │ 制动控制算法 │
│ │ (车辆动力学) │ │      │ AO: PWM反馈  │      │             │
│ └─────────────┘ │      │ DIO: 开关量  │      │ PWM输出     │
│       ↑↓        │←USB→ │              │←线束→│ 传感器输入  │
│ ┌─────────────┐ │      └──────────────┘      └─────────────┘
│ │ I/O驱动     │ │
│ └─────────────┘ │
└─────────────────┘
```

**交付物**:
- [ ] HIL硬件连线图
- [ ] Simulink车辆模型（单轮制动）
- [ ] Python/MATLAB I/O接口程序
- [ ] 基础测试场景（正常制动、紧急制动）
- [ ] 测试数据记录功能

**成功标准**:
- 闭环测试可运行 60 秒无崩溃
- 数据刷新率 > 10Hz
- 可模拟压力传感器信号反馈

**目标**: 升级为硬实时HIL平台，支持更精确的测试

**硬件升级**:
| 设备 | 型号/规格 | 成本 | 用途 |
|------|----------|------|------|
| 实时工控机 | NI PXIe-8840 或研华 | ¥30k | 硬实时主机 |

| CAN卡 | NI PXIe-8510 或 Vector | ¥10k | CAN通信 |
| 信号调理 | NI SCXI 或定制 | ¥10k | 专业级调理 |
| 负载箱 | 电子负载 + 电机模拟 | ¥20k | 执行器模拟 |

**软件栈**:
- **实时系统**: Simulink Real-Time 或 dSPACE
- **车辆模型**: CarSim 或高精度自制模型（3-DOF）

## 关键技术决策

### 1. 实时性方案选择

| 方案 | 延迟 | 复杂度 | 适用 |
|------|------|--------|------|
| Python + 裸循环 | ~10ms | 低 | MVP验证 |
| Python + RT-Preempt | ~1ms | 中 | 进阶 |
| Simulink Real-Time | <0.1ms | 中 | 标准版 |
| dSPACE | <0.01ms | 高 | 专业级 |

**建议**: MVP用Python快速验证，标准版用Simulink Real-Time

### 2. 车辆模型复杂度

| 模型 | DOF | 适用阶段 | 计算量 |
|------|-----|----------|--------|
| 单轮模型 | 1 | 算法验证 | 低 |
| 半车模型 | 4 | 动态响应 | 中 |
| 整车模型 | 15+ | 极限工况 | 高 |

**建议**: MVP用单轮，标准版用半车

### 3. I/O接口选择

| 接口 | 精度 | 速度 | 成本 |
|------|------|------|------|
| USB DAQ | 12-16bit | 中等 | 低 |
| PCI/PCIe DAQ | 16-24bit | 快 | 中 |
| PXIe DAQ | 16-24bit | 极快 | 高 |

**建议**: MVP用USB，标准版用PXIe

| ID | 风险 | 概率 | 影响 | 缓解措施 |
